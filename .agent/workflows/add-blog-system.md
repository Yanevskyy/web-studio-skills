# Blog System Implementation Workflow

Этот workflow описывает процесс добавления полнофункциональной блог-системы на сайт. Следуй шагам по порядку.

---

## Предварительные требования

- [ ] Проект на React/Next.js с react-router-dom
- [ ] Базовая структура сайта готова
- [ ] Navigation компонент с поддержкой light mode
- [ ] Изображения для статей подготовлены

---

## Шаг 1: Изучение скилла

Перед началом работы прочитай скилл blog-system:

```
view_file .agent/skills/blog-system/SKILL.md
```

Особое внимание:
- Структура данных статьи
- Паттерны реализации
- Чеклист

---

## Шаг 2: Создание страницы списка статей

Создай `src/pages/Blog.tsx`:

1. Определи массив `featuredPost` с полями: slug, title, excerpt, image, date, category, readTime
2. Определи массив `posts` с теми же полями
3. Реализуй featured post секцию (крупная карточка)
4. Реализуй grid карточек для остальных статей
5. **ВАЖНО**: Оберни все карточки в `<Link to={/blog/${slug}}>`
6. Добавь hover-эффекты через `group` и `group-hover`
7. Добавь reveal-анимации
8. Добавь Newsletter CTA секцию

// turbo

---

## Шаг 3: Создание страницы статьи

Создай `src/pages/BlogPost.tsx`:

1. Получи slug из useParams
2. Определи массив articles с полным контентом
3. Реализуй Hero секцию с:
   - Изображение на 70vh
   - Dark overlay (linear-gradient rgba(0,0,0,0.6))
   - Bottom gradient to cream
4. Добавь Back button светлого цвета
5. Реализуй контент-секцию с reveal-анимациями
6. Добавь CTA подписки
7. Добавь scroll to top в useEffect

// turbo

---

## Шаг 4: Добавление роута

Обнови `src/App.tsx`:

1. Добавь импорт: `import BlogPost from './pages/BlogPost'`
2. Добавь роут после `/blog`: `<Route path="/blog/:slug" element={<BlogPost />} />`

// turbo

---

## Шаг 5: Обновление навигации

Обнови `src/components/Navigation.tsx`:

1. Добавь проверку: `const isBlogPostPage = location.pathname.startsWith('/blog/')`
2. Обнови useLightColors: добавь `|| isBlogPostPage` в условие
3. Убедись, что логотип и меню белые на страницах статей

// turbo

---

## Шаг 6: Написание контента

Для каждой статьи:

1. Определи категорию (Our Story, Process, Philosophy, Ingredients)
2. Напиши hook — первый параграф, захватывающий внимание
3. Раздели контент на параграфы по смыслу
4. Добавь insight — ключевой вывод
5. Проверь длину: 1000-2000 слов для flagship статей

---

## Шаг 7: Тестирование

### Функциональное тестирование

```
1. Открой http://localhost:PORT/blog
2. Проверь что все карточки кликабельны
3. Кликни на featured post → должен открыться /blog/slug
4. Проверь что навигация белая на hero
5. Прокрути вниз → навигация должна стать тёмной
6. Нажми "Back to Journal" → вернуться на /blog
```

### Visual тестирование

```
1. Достаточный ли контраст текста на hero?
2. Работают ли hover-эффекты на карточках?
3. Проверь мобильную версию (320px, 375px, 768px)
4. Плавные ли reveal-анимации?
```

## Распространённые проблемы

### Навигация тёмная на статьях

Решение: Добавить `isBlogPostPage` в условие `useLightColors`

### Карточки не кликабельны

Решение: Обернуть `<article>` в `<Link>`, использовать `group` для hover

### Текст не читается на hero

Решение: Добавить dark overlay с linear-gradient

### Страница не скроллится вверх

Решение: Добавить `useEffect(() => { window.scrollTo(0, 0) }, [slug])`

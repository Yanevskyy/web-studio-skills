# Deploy Preview Workflow

## Проверка перед деплоем

// turbo
1. Выполни `npm run build` — убедись что билд проходит без ошибок

2. Если есть ошибки — исправь их

## Deploy

3. Используй скилл `vercel-deploy`

4. Если Vercel ещё не подключен:
   ```bash
   cd <project-dir>
   vercel
   ```

5. Если уже подключен:
   ```bash
   git add .
   git commit -m "Preview for client review"
   git push
   ```

## Получение URL

6. Подожди завершения деплоя (~1-2 минуты)

7. Получи preview URL:
   - Из терминала после `vercel`
   - Или из Vercel Dashboard

## Проверка

8. Открой preview URL на компьютере
9. Открой на мобильном телефоне
10. Проверь основные страницы работают

## Отправка клиенту

11. Подготовь сообщение для клиента:

```
Добрый день!

Готова preview-версия сайта для вашего review:
[URL]

Пожалуйста, посмотрите на мобильном и компьютере.
Жду ваш feedback по дизайну и контенту.
```

12. Отправь клиенту

## Ожидание

⏸️ Жди feedback. Если будут правки — вернись к `/design-phase`.
После одобрения — переходи к `/post-approval`.

# Design Phase Workflow

## Подготовка

1. Изучи текущее состояние проекта (pages, components)
2. Определи целевую аудиторию и стиль

## Mobile UX

3. Используй скилл `mobile-ux-excellence`:
   - Проверь thumb zone для кнопок
   - Найди скучные/шаблонные решения
   - Предложи wow-эффекты

4. Добавь микро-анимации:
   - Переходы между страницами
   - Hover/tap состояния
   - Scroll-triggered анимации
   - Skeleton loaders

## Design System

5. Используй скилл `design-system`:
   - Проверь tailwind.config.js
   - Унифицируй naming (semantic vs color names)
   - Проверь консистентность отступов

6. Проверь типографику:
   - Иерархия заголовков
   - Читаемость текста
   - Line-height и spacing

## Responsive

7. Используй скилл `responsive-design`:
   - Проверь все breakpoints
   - Mobile-first подход
   - Навигация на мобильном

## Навигация и структура

8. Проверь все ссылки работают
9. Логичные переходы между страницами
10. Footer одинаковый везде
11. 404 страница

## Контент

12. Все тексты на месте (не placeholder)
13. Изображения качественные и оптимизированы
14. Контактная информация актуальна

## Финальная проверка

15. Открой сайт на реальном телефоне
16. Спроси себя: "Это выглядит круто?"
17. Если нет — найди что улучшить

## Результат

Дизайн готов к показу клиенту. Перейди к `/deploy-preview`.
